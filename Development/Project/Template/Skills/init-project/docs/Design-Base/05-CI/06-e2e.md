# 06-e2e — End-to-end 測試

> **何時讀**:啟用 e2e / 改 e2e workflow 才讀。

工具:**Playwright**(跨瀏覽器 + screenshot 強)。預設 `if: false`,**不**擋 PR。

---

## 預設 disabled

新專案 e2e 風險:flaky / 慢 / 維護高。預設 workflow 帶但 `if: false`,等專案有意願啟用再開。

```yaml
# .github/workflows/e2e.yml
name: e2e
on:
  pull_request:
    branches: [main]
  workflow_dispatch:

jobs:
  playwright:
    if: false   # 改 true 啟用
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: frontend/package.json
          cache: npm
      - working-directory: ./frontend
        run: |
          npm ci
          npx playwright install --with-deps chromium
          npm run e2e
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: frontend/playwright-report/
          retention-days: 7
```

## 啟用條件

啟用 e2e 前確認:

- [ ] 有測試對象(staging 環境 URL 已穩定)
- [ ] 有 owner(誰維護 e2e suite,flaky 誰修)
- [ ] 已選定 critical user journey(登入 / 下單 / 主要 CRUD,**禁**全頁面掃)
- [ ] flaky 容忍度政策(retry 1 次 + 連 fail 2 次降級為 warning)

## 範圍

- **僅** critical journey:登入 / 主要寫入 / 付款流程
- **禁**對純 UI 樣式 / 視覺回歸 / 邊角頁面跑 e2e(改用 Vitest + RTL)
- **禁**對第三方真服務跑(用 sandbox / mock,見 `90-third-party-service/`)

## flaky 處理

- Playwright `retries: 1`(預設)
- 連 2 次 fail 同 case → 降級為 warning + 寫 `fixed.md`(根因分析)
- 連 ≥ 3 PR 都跑掛同 case → reflect 候選棄用該 case
- **禁**只為了綠加 `test.skip` 而不寫 `fixed.md`

## 對應其他規範

- 啟用後 `07-branch-protection.md` 可考慮把 e2e 加進 required check(視 flaky 程度)
- staging URL / 測試帳號用 GitHub Secrets,見 `08-secrets-and-oidc.md`
