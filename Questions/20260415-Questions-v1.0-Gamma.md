---

## 解法一：環境變數統一管理

**專案一定要有設定清單**

- 放一份 `.env.example`，只列名稱、不寫真實值
- 真正帶密碼的 `.env` 檔案**絕對不進 git**
- 程式啟動自動檢查，缺一個就立刻停掉

**所有設定不能寫死在程式裡**

- 網址、金鑰一律透過環境變數傳入
- 禁止 hardcode `localhost` / `127.0.0.1`
- 用統一的 `config` 模組集中管理

> ✅ 解決問題 1、問題 2

---

## 解法二：Docker 打包 + Git 自動部署

**用 Docker 把程式打包**

- 把程式、環境、套件裝進一個盒子
- 不管搬到哪個平台，盒子裡都完全一樣
- 「這邊能跑、那邊跑不起來」就消失了

**Coolify 走 Git 自動部署**

- push 程式碼就自動上線
- 不再手動搬檔案，也不必登入機器
- 對外網址一律使用 Coolify 分配的正式網址

> ✅ 解決問題 3

## 平台分工與當前進度

**平台怎麼分工**

| 要做什麼 | 用哪個平台 |
| --- | --- |
| 個人小工具、效率腳本 | **Replit** |
| 跨部門系統、對外服務 | **Coolify**（IT 統一管理） |
| 登入驗證 | **公司 SSO**（單一登入） |

**當前進度**

- ✅ 企業統一 SSO 登入入口
- ✅ Coolify 部署格式規範
- 🟡 Coolify 操作 Web App（簡化部署 UI）
- 🟡 企業專案格式規範

> 💡 核心原則：設定不寫死、密碼不進 git、有錯就立刻停
